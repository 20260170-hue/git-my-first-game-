# 4주차 실습 정리
## 오늘 한 것
원형 / AABB / OBB Bounding Box를 직접 구현하고 시각화함
세 가지 충돌 방식(Circle, AABB, OBB)을 동시에 비교함
회전하는 오브젝트를 통해 충돌 판정 차이를 확인함
## 관찰 내용
### 원형 vs AABB
원형 충돌은 중심 거리만으로 계산하기 때문에 매우 단순하고 빠름
하지만 사각형 오브젝트에서는 실제보다 더 크게 충돌이 발생하는 경우가 있음
→ 모서리 부분에서 아직 닿지 않았는데도 충돌로 판정됨
AABB는 사각형 기준이라 원형보다 더 정확하지만, 회전이 없다는 전제에서만 정확함

결론:

원형 → 빠르지만 부정확
AABB → 원형보다 정확하지만 제한적
AABB vs OBB
AABB는 항상 축에 정렬되어 있어서 회전한 오브젝트를 정확히 반영하지 못함
→ 회전 시 박스가 실제보다 커지거나 작아지는 문제 발생
OBB는 오브젝트의 회전을 그대로 반영하기 때문에
 가장 정확한 충돌 판정 가능
특히 SAT(분리축 정리)를 사용하면
→ 두 도형이 실제로 겹치는지 정밀하게 판단 가능

### AABB → 빠르지만 회전에 약함
### OBB → 가장 정확하지만 계산 비용이 큼
## AI와의 대화에서 배운 것
OBB 충돌은 단순 비교가 아니라 SAT(분리축 정리)를 이용해 판정한다는 점이 인상적이었음
하나의 축이라도 겹치지 않으면 충돌이 아니라는 개념이 흥미로웠음
또한 실제 게임에서는
AABB로 1차 필터링 후 OBB로 정밀 검사하는 방식이 효율적이라는 것을 배움
## 내 게임에 적용한다면
기본적으로는 AABB 충돌을 사용할 것 같음
→ 이유: 구현이 쉽고 성능이 빠르기 때문
대신 중요한 오브젝트(캐릭터, 보스 등)는
OBB 또는 정밀 충돌 방식 추가 적용

일반 오브젝트 → AABB
중요한 오브젝트 → OBB (또는 정밀 충돌)
